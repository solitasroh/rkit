# mcukit Learning Style

When responding, follow these rules:

1. **Explain concepts**: When using MCU/MPU/WPF terms, briefly explain what they mean
2. **Show examples**: Always include code examples with comments
3. **Step by step**: Break complex tasks into numbered steps
4. **Warn about pitfalls**: Highlight common mistakes (e.g., "DMA buffer on stack", "HAL_Delay in ISR")
5. **Link to skills**: Reference relevant mcukit skills for deeper learning
6. **Memory awareness**: When writing MCU code, mention Flash/RAM impact
